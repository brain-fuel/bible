# L2b Lemma Relation-Graph Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a many-valued lemma relation-graph (domain-sibling, shared-root, cross-language, synonym, antonym) where every edge carries provenance + an integer `rank` (0..65535), materialized as canonical JSONL and projected into a queryable DB `relations` table.

**Architecture:** Two-form, additive. Builders read canonical `lexicon/**/*.json` (domains/root/glosses) + the LXX `mt_lxx` bridge + verified PD/CC-BY thesauri, and emit canonical line-oriented JSONL edge files under `relations/`. A per-source normalizer/miner quarantines source-format messiness behind a fixed edge schema; everything downstream is deterministic. `build_db` projects all JSONL into a `relations` table with a default rank-threshold view. Nothing is dropped — low-quality edges get a low `rank`.

**Tech Stack:** Python 3 stdlib only (`json`, `sqlite3`, `hashlib`, `math`, `unicodedata`, `re`, `xml.etree`, `urllib`, `pathlib`), `pytest`. No new runtime deps. Reuses floor/LXX tools (`tools.align_mt_lxx.build_bridge`, `lexicon/`, `tools.build_db`).

## Global Constraints

- **Many-valued, never capped:** no relation is ever limited to one per word. A lemma keeps every domain-sibling, every shared-root cognate, every origin, every synonym/antonym. Filtering is by `rank` only, never by a per-word count.
- **Rank, not drop:** every derivable edge is materialized and committed. The 16-bit unsigned `rank` (`0..65535`, `2^16-1`) is the single quality filter/sort key. `DEFAULT_RANK_THRESHOLD` (a pinned constant) defines the "bounded default" DB view; lowering it to 0 returns everything. The threshold filters quality, never a word's edge count.
- **Lemma keys:** edge endpoints are existing lexicon keys — `G####` / `H####` (Strong's) or `lemma-<sha1(lemma)[:12]>` (the null-strong LXX-only slug already used in `lexicon/grc/lemma-*.json`). Cross-language edges connect `H####`↔`G####`.
- **Provenance on every edge:** `provenance.method` ∈ {authored, derived, projection, mined}; `provenance.source` names the exact canon/calc. Every edge has `rank` ∈ `0..65535`.
- **Two-form:** `relations/authored/<rel>.jsonl` = hand CC0 truth (never overwritten by a builder). `relations/derived/<rel>.jsonl` = regenerated by builders (committed, rebuildable). DB `relations` table = projection of ALL of them. Builders only ever write under `relations/derived/`.
- **Licensing (per `docs/LICENSING.md`):** builder code = AGPL-3.0; authored overlay = CC0; derived edges inherit the most-restrictive license of their inputs (Louw-Nida/SDBH/`mt_lxx` → CC-BY; Roget/Abbott-Smith/BDB/Lewis&Short/Strong's → PD/CC0; Open English WordNet → CC-BY). `provenance.source` makes each edge's license traceable. Raw mining downloads live under gitignored `data/cache/`.
- **Encoding:** raw lemma strings (NO NFC on lemma keys — consistent with the floor's oxia join). JSONL UTF-8, LF, one edge per line.
- **Determinism:** every builder is idempotent; derived JSONL sorts by `(src, rel, dst, source)` and rebuilds byte-identical; the DB rebuilds with stable counts.
- **No self-loops; symmetric storage:** `src != dst`. Symmetric relations (all but cross-language are symmetric; cross-language stored once per H,G with `directed:false`) are stored canonically (emit one orientation, the loader/queries treat them as undirected) — pick and document the canonical orientation (e.g. `src < dst` lexicographically) so regen is stable.
- **L0/floor immutability:** never modify `bible/`, `morph/`, or existing `lexicon/**` entries. `relations/` is a NEW tree. TDD: failing test first, minimal code, commit per task.

## Edge schema (the fixed contract every builder emits)

```json
{"src":"G0026","dst":"G0025","rel":"shared-root","directed":false,
 "provenance":{"source":"strongs-root","method":"derived"},"rank":65535,"note":null}
```

`rel` ∈ {domain-sibling, shared-root, cross-language, synonym, antonym}. `rank` uint16. `note` optional string or null.

### Task 6: gloss-bridge miner engine (shared)

**Files:** Create `tools/relations/gloss_bridge.py`, `tests/test_gloss_bridge.py`.

**Interfaces:**
- Consumes: `lexicon/**/*.json` `glosses` (lang-keyed); `Edge`, `rank`.
- Produces:
  - `gloss_term_index(lang="en") -> dict[str, set[str]]` — maps a normalized gloss TERM → the set of lemma keys whose `glosses[lang]` contains that term. (Tokenize gloss text into terms; lowercase; strip Strong's cross-refs like `from G0025`; drop stopwords.)
  - `mine_relations(headword_links, lang, source, method, base_rank) -> list[Edge]` — given a thesaurus's `headword_links: Iterable[(headword:str, related:str, rel:str)]` (rel ∈ {synonym, antonym}), map `headword`→lemma keys and `related`→lemma keys via `gloss_term_index(lang)`, and emit a `rel` edge for each (key_a, key_b) cross-product, `source`/`method` as given, `rank = base_rank` lowered by a match-looseness penalty (e.g. multi-word/partial gloss match → lower). Symmetric → canonical orientation; no self-loops; endpoints validated.
  This is the SHARED engine Tasks 7-9 call; each of those only parses its source into `headword_links` and picks `source`/`base_rank`.
